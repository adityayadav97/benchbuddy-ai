About BenchBuddy AI

Built for the EngX Generative AI Kata — 19 Jun 2026.

What it does

  • Accepts free-text associate queries.
  • Retrieves grounding rows from the PMO FAQ knowledge base.
  • Returns answer, confidence score, category and status.
  • Escalates time-sensitive / negative-sentiment / out-of-knowledge queries to PMO.
  • Never hallucinates — every answer is composed only from KB rows.

Tech stack

  • Backend: Python · FastAPI
  • Retrieval: TF-IDF word + char n-grams · cosine
  • Frontend: Vanilla HTML / CSS / JS · zero build step
  • Knowledge: — FAQs

Guard-rails

  • Anti-hallucination: answers verbatim from KB.
  • No external knowledge: no web, no LLM world-model.
  • No approval / staffing decisions: always escalates to PMO.
  • Out-of-scope detection: polite refusal with OutOfScope.
  • Sentiment safety net: negative messages auto-escalate.
  • Multi-intent splitter: long messages handled cleanly.

JSON contract

POST /api/query with { "query": "..." }:

{
  "answer": "...",
  "confidence": 0.87,
  "confidence_pct": 87,
  "category": "Certification",
  "status": "Answered",
  "escalation_target": null,
  "sources": [{"id": 26, "rank": 1, ...}],
  "intents_detected": ["Certification"],
  "sentiment": "neutral",
  "latency_ms": 4
}

API endpoints

  • POST /api/query — ask a question
  • GET /api/faqs — full KB dump
  • GET /api/kb/{id} — one FAQ by id
  • GET /api/related/{id} — related FAQs
  • GET /api/categories — KB taxonomy + counts
  • GET /api/analytics — session analytics
  • POST /api/escalations — submit to PMO
  • GET /api/escalations — ticket list
  • GET /api/health — liveness probe
  • GET /api/samples — sample queries
